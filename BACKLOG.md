# Tech Debt & Backlog

Generic CRM (обычные CRM) плохи для бухгалтеров, потому что там нет главного: цикличности и автоматизации рутины. Бухгалтерия — это не про «воронку продаж» (позвонил → продал), это про бесконечный цикл: «Квартал закрыт → Сдать НДС → Сдать Прибыль → Повторить».

Система проектируется и разрабатывается под реалии бухгалтерского аутсорса Республики Беларусь, с заделом на лёгкое масштабирование/адаптацию под РФ/Казахстан/СНГ.

---

## Killer Features
*   [ ] Конвертер присланных клиентом фото/сканов/прочих документов (JPG/PNG/PDF/TXT/etc → 1C/Excel).
```
Проблема: Клиенты часто присылают выписки из банка в формате PDF (или сканы). Чтобы загрузить их в 1С, бухгалтеру нужно либо перебивать руками, либо конвертировать их в специальный текстовый формат (kl_to_1c.txt). Это боль.
Решение: Веб-сервис или десктоп-приложение.
Бухгалтер кидает PDF-файл.
Python (pdfplumber или camelot) вытаскивает таблицы.
Скрипт формирует файл, который 1С «съедает» без ошибок.
Стек: Python, Pandas, Streamlit (для веб-интерфейса).
```
*   [ ] Автоматическая «Сверка» (Reconciliation).
```
Проблема: Самая ненавистная работа. Есть таблица «Наши данные» и таблица «Данные поставщика». В каждой по 1000 строк. Нужно найти, какие суммы не сходятся (кто-то потерял документ, где-то ошиблись в копейках). Глазами это делать — часы работы.
Решение: Скрипт интеллектуальной сверки.
Загружаем два Excel-файла (даже если у них разные колонки и форматы).
Python (Pandas) ищет совпадения по дате, сумме и номеру документа.
Выдает результат: «Вот эти 5 позиций не найдены, а вот тут сумма отличается на 2 копейки».
Фишка: Можно добавить «нечеткое сравнение» (Fuzzy Matching), чтобы понимать, что «ООО Ромашка» и «Ромашка ООО» — это одно и то же.
Стек: Python, Pandas, RapidFuzz.
```
*   [ ] Парсер отчетов Маркетплейсов (Wildberries / Ozon) для 1С.
```
Проблема: Сейчас бум селлеров. Отчеты о продажах WB — это гигантские Excel-таблицы. Заносить их в 1С — ад, потому что нужно правильно разнести комиссии, логистику, штрафы и возвраты по счетам бухучета. Штатная 1С часто не поспевает за изменениями форматов отчетов WB.
Решение: Специализированный загрузчик.
Скрипт берет еженедельный отчет WB (Excel/API).
Разбирает его на составляющие (продажи, возвраты, логистика).
Генерирует файл загрузки для 1С или сразу создает документы в 1С через OData/COM-соединение.
Деньги: Это B2B продукт. Аутсорсинговые фирмы, ведущие селлеров, за такое душу продадут.
Стек: Python, OpenPyXL, Requests (API WB).
```
*   [ ] Telegram-бот для сбора «Первички» (чеков и накладных).
```
Проблема: Бухгалтеры тратят уйму времени, выпрашивая у клиентов чеки: «Иван Иванович, вы вчера корпоративной картой оплатили бензин, пришлите чек!». Иван Иванович забывает, чек теряется.
Решение: Бот-архивариус.
Клиент (директор фирмы) сфоткал чек на кассе и сразу кинул в бота.
Бот (Python + OCR, например Yandex Vision или Tesseract) распознает дату и сумму.
Сохраняет фото в папку на Google Диске / Яндекс.Диске с именем 2023-10-05_1500руб_Бензин.jpg.
Бухгалтер в конце месяца просто открывает папку, где всё уже лежит красиво.
Стек: Aiogram, Google Drive API, OCR libraries.
```
*   [ ] Робот-проверщик контрагентов (Due Diligence).
```
Проблема: Перед оплатой счета нужно проверить, не банкрот ли фирма, нет ли у неё судов, не «помойка» ли это. Сайтов много (ФНС, КадАрбитр, Федресурс), лазить по ним долго.
Решение: Агрегатор рисков.
Бухгалтер вводит ИНН.
Скрипт лезет на сайты налоговой, судов (через API или парсинг).
Генерирует PDF-отчет: «Риски высокие: идет процедура банкротства, директор массовый».
Стек: Python, Selenium/BeautifulSoup, API (DaData, Контур.Фокус — если есть бюджет, или парсинг открытых источников).
```
